프로젝트 📱

\- Commit

​	- 커밋단위 : 빌드되는 상태에서, 기능단위로 커밋하기.

​	- Type 규칙 (커밋 메세지에 이슈 번호를 추가하면 해당 이슈로 commit이 넘어감)

​		- 예시 → Type종류 #이슈번호 - commit 내용

​		- Feat: 기능 구현

​		- Refactor: 기능 수정

​		- Fix: 버그 수정

​		- Style: 동작에 영향없는 코드변경, 이름변경

​		- Docs: 문서수정

​		- Chore: 기타



\- 코딩 컨벤션 👀

​	- naming에 신경쓰기.

​	- 의미없는 띄어쓰기 안하도록 주의하기.

​	- 기능 분리 잘 되도록 주의하기. (1 함수 1 기능 지향)
